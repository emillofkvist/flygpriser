1. VAD VI BYGGER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

En interaktiv 3D-visualisering av Helsingborgs busstrafik i realtid.
Portfolio-showcase med cinematisk "levande stad"-estetik.

Visuellt koncept:
  - Mörk karta med Helsingborgs riktiga 3D-byggnader (extruderade från OSM-data)
  - Bussrutter visas som glödande neon-tuber som svävar ovanför gatunivå
  - Bussar i realtid visas som lysande guldgula partiklar längs rutterna
  - Hållplatser visas som pulserande cyan-ljuspelare (höjd = trafiktäthet)
  - Kameran roterar långsamt automatiskt, stannar vid interaktion
  - Klicka på en rutt för att se linjeinformation i overlay

3. TEKNIKSTACK
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  React 19 + Vite 8      — Frontend-ramverk och byggverktyg
  deck.gl 9              — GPU-accelererade 3D-kartlager (Uber/Vis.gl)
  MapLibre GL            — Open source kartmotor (OpenGL-baserad)
  react-map-gl           — React-wrapper för MapLibre
  gtfs-realtime-bindings — Parsar Protobuf-data från Trafiklab
  JSZip                  — Packar upp GTFS-static ZIP-filer i webbläsaren

deck.gl-lager som används:
  PathLayer     — Bussrutter som 3D-linjer (3 lager per rutt = glow-effekt)
  ScatterplotLayer — Bussar (realtid + animerade demo-bussar + trail)
  ColumnLayer   — Hållplatser som cylindrar

Node.js krävs: https://nodejs.org (LTS-versionen rekommenderas)

7. HUR APPEN FUNGERAR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Vid start:
  1. Kartan laddas med Helsingborg i centrum, 3D-byggnader extruderas
  2. Demo-bussar börjar animeras direkt (vita prickar längs rutterna)
  3. Trafiklab Static laddas i bakgrunden → riktiga rutter och hållplatser
  4. Trafiklab RT pollas → riktiga bussar visas som guldgula prickar
  5. Status-badge i övre vänstra hörnet visar "X bussar live"

Interaktion:
  - Dra musen: rotera/panorera kartan (autorotation pausas)
  - Scroll: zooma in/ut
  - Klicka på en rutt: linjeinformation visas i overlay
  - Klicka "Avmarkera": stänger overlay

Data-flöde:
  - GTFS Static (rutter/hållplatser): hämtas EN gång, cachas 23 timmar
  - GTFS-RT (busspositioner): hämtas var 15:e sekund
  - Vite-proxy i dev-läge hanterar CORS mot Trafiklab
